# SwimHub Timer React Native 移行 要件定義書

## 1. プロジェクト概要

### 1.1 目的

既存のNext.js Webアプリ「SwimHub Timer」をReact Nativeアプリとして iOS / Android 向けにリリースする。

### 1.2 アプリ概要

水泳レース動画にストップウォッチオーバーレイを自動追加するビデオ編集アプリ。
3ステップのワークフロー: **動画取り込み → スタート合図検出＆デザイン → 書き出し**

### 1.3 対象プラットフォーム

| プラットフォーム | 最低バージョン            |
| ---------------- | ------------------------- |
| iOS              | 15.0 以上                 |
| Android          | API 26 (Android 8.0) 以上 |

## 2. 現行Web版の技術スタック

| カテゴリ       | 技術                                | RN移行での課題             |
| -------------- | ----------------------------------- | -------------------------- |
| フレームワーク | Next.js 16 (Static Export)          | RN に置き換え              |
| UI             | shadcn/ui + Radix UI + Tailwind CSS | RN用UIライブラリに移行     |
| 状態管理       | Zustand                             | **そのまま利用可能**       |
| 動画処理       | FFmpeg WASM (ブラウザ内)            | ネイティブFFmpegに変更     |
| 音声解析       | Web Audio API                       | ネイティブ音声処理に変更   |
| 描画           | Canvas 2D API                       | RN Canvas / Skia に変更    |
| ストレージ     | IndexedDB (idb-keyval)              | AsyncStorage / MMKV に変更 |
| 型定義         | TypeScript                          | **そのまま利用可能**       |

## 3. 機能要件

### 3.1 動画取り込み (Import)

| ID     | 要件                                           | 優先度 | 備考                         |
| ------ | ---------------------------------------------- | ------ | ---------------------------- |
| IMP-01 | デバイスのフォトライブラリから動画を選択できる | 必須   | react-native-image-picker 等 |
| IMP-02 | カメラロールからの直接撮影・取り込みに対応する | 推奨   |                              |
| IMP-03 | 対応フォーマット: MP4, MOV                     | 必須   |                              |
| IMP-04 | 動画メタデータ(解像度・尺)を取得・表示する     | 必須   |                              |
| IMP-05 | ファイルサイズ制限の案内を表示する             | 推奨   | デバイスメモリに依存         |

### 3.2 スタート合図検出 (Signal Detection)

| ID     | 要件                                         | 優先度 | 備考                   |
| ------ | -------------------------------------------- | ------ | ---------------------- |
| SIG-01 | 動画から音声を抽出し波形を表示する           | 必須   | ネイティブ音声デコード |
| SIG-02 | 波形をタップして任意の時間を選択できる       | 必須   |                        |
| SIG-03 | 電子スタート音を自動検出する (STFT解析)      | 必須   | 既存アルゴリズムを移植 |
| SIG-04 | 検出結果を ±100ms / ±10ms 単位で微調整できる | 必須   |                        |
| SIG-05 | スタート時刻を確定する操作がある             | 必須   |                        |

### 3.3 ストップウォッチデザイン (Stopwatch Designer)

| ID     | 要件                                            | 優先度 | 備考                                  |
| ------ | ----------------------------------------------- | ------ | ------------------------------------- |
| STW-01 | 4種のプリセットスタイルを選択できる             | 必須   | Classic/Minimal/Broadcast/Competition |
| STW-02 | フォントサイズを 20〜80px で調整できる          | 必須   |                                       |
| STW-03 | ストップウォッチの表示位置を9箇所から選択できる | 必須   |                                       |
| STW-04 | ドラッグで自由配置できる                        | 必須   |                                       |
| STW-05 | リアルタイムプレビューを表示する                | 必須   |                                       |

### 3.4 動画プレビュー & 再生 (Video Preview)

| ID     | 要件                                               | 優先度 | 備考 |
| ------ | -------------------------------------------------- | ------ | ---- |
| VID-01 | 動画をストップウォッチオーバーレイ付きで再生できる | 必須   |      |
| VID-02 | 再生/一時停止/シーク操作ができる                   | 必須   |      |
| VID-03 | アスペクト比を維持したレターボックス表示           | 必須   |      |
| VID-04 | シークバーで任意の位置にジャンプできる             | 必須   |      |

### 3.5 スプリットタイム記録 (Splits)

| ID     | 要件                                                | 優先度 | 備考 |
| ------ | --------------------------------------------------- | ------ | ---- |
| SPL-01 | 任意の距離(m)でスプリットタイムを記録できる         | 必須   |      |
| SPL-02 | ラップタイム(50mごと)を自動計算する                 | 必須   |      |
| SPL-03 | 各スプリットにメモを追加できる                      | 必須   |      |
| SPL-04 | フィニッシュタイムを記録できる                      | 必須   |      |
| SPL-05 | スプリットをタップで該当位置にシークする            | 必須   |      |
| SPL-06 | 個別スプリットを削除できる                          | 必須   |      |
| SPL-07 | 書き出し時に3秒間のスプリットオーバーレイを表示する | 必須   |      |

### 3.6 動画書き出し (Export)

| ID     | 要件                                         | 優先度 | 備考                             |
| ------ | -------------------------------------------- | ------ | -------------------------------- |
| EXP-01 | 720p / 1080p / オリジナル画質で書き出せる    | 必須   |                                  |
| EXP-02 | H.264 エンコード (MP4) で出力する            | 必須   | ネイティブFFmpeg                 |
| EXP-03 | リアルタイムで進捗バーを表示する             | 必須   |                                  |
| EXP-04 | 書き出し完了後にフォトライブラリへ保存できる | 必須   |                                  |
| EXP-05 | 書き出し完了後に共有シートで共有できる       | 推奨   | iOS Share Sheet / Android Intent |
| EXP-06 | バックグラウンド処理で書き出しを継続する     | 推奨   | 長時間動画対応                   |

### 3.7 モバイル固有機能

| ID     | 要件                                             | 優先度 | 備考                      |
| ------ | ------------------------------------------------ | ------ | ------------------------- |
| MOB-01 | フォトライブラリのアクセス権限を管理する         | 必須   | iOS/Androidパーミッション |
| MOB-02 | アプリアイコン・スプラッシュスクリーンを設定する | 必須   |                           |

## 4. 非機能要件

### 4.1 パフォーマンス

| ID     | 要件                           | 目標値                     |
| ------ | ------------------------------ | -------------------------- |
| PER-01 | アプリ起動時間                 | 2秒以内 (コールドスタート) |
| PER-02 | 動画プレビューのフレームレート | 30fps以上                  |
| PER-03 | スタート音検出処理時間         | 10秒以内 (3分動画)         |
| PER-04 | 動画書き出し速度               | リアルタイム以上 (1080p)   |
| PER-05 | メモリ使用量                   | 500MB以下 (編集中ピーク)   |

### 4.2 信頼性

| ID     | 要件                                                   |
| ------ | ------------------------------------------------------ |
| REL-01 | 書き出し中のアプリクラッシュ時にデータが失われない     |
| REL-02 | 大容量動画(1GB超)でもOOMが発生しない                   |
| REL-03 | エラー発生時に適切なメッセージを表示しリカバリーできる |

### 4.3 セキュリティ & プライバシー

| ID     | 要件                                                |
| ------ | --------------------------------------------------- |
| SEC-01 | 動画データはデバイス内のみで処理し外部送信しない    |
| SEC-02 | 最小限の権限のみ要求する (フォトライブラリ、カメラ) |
| SEC-03 | App Tracking Transparency は不要 (トラッキングなし) |

## 5. 技術設計方針

### 5.1 推奨技術スタック

| カテゴリ         | 技術                                       | 選定理由                                 |
| ---------------- | ------------------------------------------ | ---------------------------------------- |
| フレームワーク   | **Expo (SDK 52+)**                         | 開発効率・EAS Build・OTAアップデート     |
| ナビゲーション   | **Expo Router**                            | ファイルベースルーティング (Web版と類似) |
| UI               | **NativeWind (Tailwind for RN)**           | 既存Tailwindクラスを最大限再利用         |
| 状態管理         | **Zustand**                                | 既存コード完全流用                       |
| 動画再生         | **expo-video**                             | Expo公式、高パフォーマンス               |
| 動画処理         | **ffmpeg-kit-react-native**                | ネイティブFFmpeg (WASM不要)              |
| 音声解析         | **expo-av + カスタムネイティブモジュール** | 音声抽出・波形データ取得                 |
| 描画             | **@shopify/react-native-skia**             | Canvas互換の高性能2D描画                 |
| ストレージ       | **react-native-mmkv**                      | 高速KVストア (IndexedDB代替)             |
| 画像/動画選択    | **expo-image-picker**                      | フォトライブラリアクセス                 |
| ファイルシステム | **expo-file-system**                       | 一時ファイル管理                         |
| 共有             | **expo-sharing**                           | OS共有シート                             |
| メディア保存     | **expo-media-library**                     | フォトライブラリ保存                     |

## 6. 主要リスクと対策

| リスク                           | 影響度 | 対策                                                                     |
| -------------------------------- | ------ | ------------------------------------------------------------------------ |
| FFmpeg WASMがRNで動作しない      | 高     | ffmpeg-kit-react-native を使用。ネイティブバイナリで高速処理             |
| Web Audio APIが使えない          | 高     | expo-av で音声抽出後、JS側でSTFT解析。必要に応じネイティブモジュール作成 |
| Canvas APIが使えない             | 高     | react-native-skia で同等の描画を実現                                     |
| 動画上のリアルタイムオーバーレイ | 中     | Skia + expo-video の組み合わせでレイヤー合成                             |
| 大容量動画のメモリ不足           | 中     | ストリーム処理、一時ファイル活用でメモリ制御                             |
| App Store / Play Store 審査      | 中     | ガイドライン遵守、プライバシーポリシー準備                               |
| 音声波形のインタラクティブ表示   | 中     | Skia Canvasでカスタム描画、ジェスチャーハンドラ使用                      |
